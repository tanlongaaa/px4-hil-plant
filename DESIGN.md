## 1. 系统架构

### 1.1 三层分层控制

```
┌──────────────────────────────────────────────────────────────┐
│  L3  RL 策略 (Cost-Weight Scheduler)          ~1–5 Hz (低频)   │
│      输入: 风估计 ŵ, 姿态, 位置误差, 功耗, 湍流强度σ̂           │
│      输出: MPC 代价权重 θ = {Q_xy, λ_energy, R_ω, ...}         │
│      训练: PPO (MPC 当黑盒) → 后续可升级 differentiable MPC     │
└───────────────────────────────┬──────────────────────────────┘
                                 │ θ (cost weights)
                                 ▼
┌──────────────────────────────────────────────────────────────┐
│  L2  MPC (Trajectory-Coherent Wind-Adaptive MPC)   50 Hz      │
│      模型: 10D CTBR 刚体 + 风扰 + BEMT 能耗                     │
│      求解: iLQR / OSQP, 含约束 (推力/角速率/倾角)               │
│      输出: CTBR = [f_c, ωx, ωy, ωz]  (总推力 + 体轴角速率)      │
│      角色: 安全滤波器 — 保证可行性与约束, RL 永不直接碰执行器   │
└───────────────────────────────┬──────────────────────────────┘
                                 │ AttitudeTarget (setpoint_raw/attitude)
                                 ▼
┌──────────────────────────────────────────────────────────────┐
│  L1  PX4 底层 (角速率/姿态 PID + 混控器)        ~250–1000 Hz   │
│      吸收: 电机滞后, 高频扰动                                   │
│      已标定: MPC_XY_VEL_I_ACC=1.2 等 (抗风调参完成)            │
└───────────────────────────────┬──────────────────────────────┘
                                 │ motor outputs (0~1) / 实机 PWM/DShot
                                 ▼
┌──────────────────────────────────────────────────────────────┐
│  Plant  仿真: plant_6dof (HIL) ──┐    实机: 真实四旋翼         │
│         风场: wind_field (Dryden)│    ESC RPM 遥测 → 功耗      │
│         功耗: BEMT (电机指令→RPM→P)                            │
└──────────────────────────────────────────────────────────────┘
```

### 1.2 分层理由（论文 justification）

| 层 | 频率 | 为什么是它 |
|----|------|-----------|
| RL 调权重 | 低频 | 安全（MPC 兜底约束）、可解释、易部署、对实机鲁棒；避免高频 RL 直接控制的安全风险 |
| MPC | 50 Hz | 在线重规划吸收可预测扰动；显式约束保证可行性；代价权重是自然的"行为旋钮" |
| PX4 PID | 高频 | 吸收电机滞后与不可建模高频；工业级成熟、直接对应实机 |

**核心论点（contribution）：**
> 现有抗风控制几乎都以**跟踪精度**为目标。本工作提出**能耗优先**的视角：
> RL 学习**风况自适应的 MPC 代价权重**，在「能耗 vs 精度」Pareto 前沿上，
> 依据实时风态选择能耗最优工作点。MPC 保证安全可行，RL 只在外层调度。

### 1.3 与 ACMPC 的区别（novelty 定位）

| 维度 | ACMPC (TRO 2025) | 本工作 |
|------|------------------|--------|
| RL 学习对象 | 端到端策略 + 可微 MPC | **MPC 代价权重调度** |
| 优化目标 | 跟踪精度 | **能耗优先** + 精度约束 |
| 仿真保真度 | 纯 PyTorch 刚体, 无电机/气动 | **HIL plant_6dof + 真 PX4** (含风扰/旋翼气动) |
| 底层 | Agilicious 角速率 PID (闭源) | **PX4 PID (开源, 已标定)** |
| 能耗模型 | 无 | **BEMT, sim/real 统一** |
| 部署 | 实机 | 实机 + sim-to-real gap 量化 |

### 1.4 控制器模型与被控对象的有意失配（方法论辩护 — 审稿人答辩）

> 本节专门回答审稿人必问的问题：**为什么 MPC 内部用简化的 10D CTBR 模型，
> 而不用与被控对象 plant_6dof 同款的完整 6-DOF 方程？**

#### 1.4.1 两套模型，两个目的（不是同一个东西）

| 模型 | 文件 | 颗粒度 | 设计目标 |
|------|------|--------|---------|
| **被控对象 (plant)** | `plant_6dof.py` | 完整 6-DOF：刚体平动+转动、转子侧向阻力、线性/角阻尼、叶片挥舞、风注入（Iris 标定） | **最大化保真度** → 当“真值”用，评测才诚实 |
| **MPC 预测内模** | `quadrotor_dynamics_10d` | 简化 10D CTBR：`[p,q,v]` + `[f_c,ω]`，姿态动力学交给内环 | **控制导向**：够用前提下最小复杂度 → 50Hz 实时可解 |
| **真飞控** | PX4 EKF2+级联PID+混控 | 工业级全栈 | 吸收电机滞后/高频，对应实机 |

**关键认知：控制器内模 ≠ 真实 plant 是反馈控制的常态，不是缺陷。**
若 MPC 内模 = 真实 plant，则无需反馈（开环规划即可）。鲁棒/自适应控制的存在前提
就是 model-plant mismatch 必然存在。问题不在“两套模型不同”，而在**简化得对不对 + 是否诚实交代**。
用高保真模型当 MPC 内模反而错：求解变慢、过拟合到具体参数、对部署无益。

#### 1.4.2 三条理论依据（论文须引文献写明）

1. **关注点分离 (control-oriented modeling)**：plant 求保真当真值，MPC 求精简保实时，
   两者目标正交，不应统一。
2. **时间尺度分离 / 奇异摄动**：PX4 内环 250–1000Hz ≫ MPC 50Hz，从 MPC 视角
   “命令一个 body rate，内环近似瞬时跟上”。电机滞后/完整转动动力学是快内环的职责，
   塞进 MPC 是浪费。（级联控制奇异摄动理论）
3. **微分平坦性**：四旋翼在 [位置, 偏航] 微分平坦，CTBR（总推力+体轴角速率）是天然
   平坦输出兼容接口。引 Mellinger & Kumar 2011、Faessler et al. 2018。

**部署现实论据（最硬）**：真机上本来就是把 CTBR 发给 PX4，MPC 从不直接驱动电机。
让 MPC 内模颗粒度 = 实际作动接口（CTBR）是最诚实的匹配；建模超出 CTBR 层对部署无益。

**定位**：本分层（plant 完整 6-DOF / MPC 简化 CTBR / PX4 真飞控）与顶刊主流一致
（ACMPC TRO2025、Falanga、Sun 等 RAL）——MPC 层普遍是 CTBR 简化模型。本工作在主流队伍内。

#### 1.4.3 失配的代价：次优而非失稳（可接受性论证）

只要 (a) 内环稳定、(b) MPC 滚动重规划 (receding horizon) 持续纠偏、
(c) 诚实报告 gap，则失配仅导致“MPC 规划在真实 plant 上非严格最优”——可接受。
审稿人不会因 MPC 内模简化而拒稿。

**交付证据（实验须给）**：
- sim-to-real gap（实机 vs sim 的 E_total/误差偏差 %）
- *(加分)* MPC-model-to-plant gap：在 plant 上跑、对比 MPC 内部预测轨迹偏差，
  证明滚动重规划+内环吸收了失配
- 极端湍流档（§6.3 sigma_scale=1.0）实验**不只测性能，更是为简化模型有效性辩护**：
  证明即使气动耦合在极端工况下变强，跟踪/能耗仍有界且优于 baseline。

## 2. 问题形式化定义

### 2.1 控制对象（被控系统）

四旋翼 10D CTBR 状态（与现有 `quadrotor_dynamics_10d` 对齐）：

```
x = [p(3), q(4), v(3)] ∈ R^10     # 位置(ENU) + 姿态四元数 + 速度
u = [f_c, ωx, ωy, ωz] ∈ R^4       # 总推力(N) + 体轴角速率指令 (CTBR)
```

风扰 `w ∈ R^3`（ENU 风速），经 plant 转子级 + 机身级气动注入。

### 2.2 RL 问题（外层 MDP）

| 元素 | 定义 | 说明 |
|------|------|------|
| **观测 o_t** | `[ê_pos(3), v(3), tilt(2), ŵ(3), σ̂_turb(1), P_est(1), θ_{t-1}]` | 风估计 + 误差 + 功耗 + 上一步权重 |
| **动作 a_t** | `θ_t = [Q_xy, Q_z, λ_energy, R_ω]`（归一化到有界区间） | MPC 代价权重子集 |
| **奖励 r_t** | 见 §2.3 | 能耗优先的复合奖励 |
| **频率** | 1–5 Hz（每 N 个 MPC 步更新一次 θ） | 低频调度 |
| **回合** | 一次定点抗风飞行（固定时长 T，固定风 seed） | |

### 2.3 奖励函数（能耗优先）

```
r_t = - α_E · (E_t / E_hover)              # 能耗项 (主) — 归一化到悬停能耗
      - α_p · ‖e_pos‖² · 1[‖e_pos‖ > δ]    # 位置误差 (仅超出容差 δ 才罚)
      - α_c · (constraint_violation)         # 约束违反 (硬惩罚)
      - α_s · ‖θ_t − θ_{t-1}‖²              # 权重平滑 (防抖动/振荡)
```

设计要点：
- **能耗优先**通过 `α_E ≫ α_p` 体现，且位置误差用**死区**（容差 δ 内不罚）→ 鼓励"够用就好"
- 约束违反走 MPC 硬约束 + 奖励软惩罚双保险
- 平滑项防 RL 高频抖权重激发系统振荡

> ⚠️ 待定参数：α_E, α_p, α_c, α_s, δ, E_hover 基准。
> 这些是**奖励塑形**关键，需在 §6 实验中通过敏感性分析确定，并在论文中报告。

### 2.4 约束（MPC 硬约束，安全边界）

```
f_c ∈ [2, 20] N              # 总推力 (10D nominal, 见现有代码)
|ωx|, |ωy| ≤ 0.1 rad/s       # 体轴角速率 (CTBR, 极小, 防超调)
|ωz| ≤ 0.05 rad/s
tilt ≤ MPC_TILTMAX_AIR=45°   # 倾角 (PX4 层)
```

## 3. MPC 接口规范（重设计要求）

> 这是 MPC 重设计的**验收标准**。重设计前必须满足以下接口，否则 RL 无法接入。

### 3.1 现状（已有，offboard_test/scripts/）

现有 `mpc_node.py` 已是 10D CTBR iLQR，且**已有权重钩子**（关键发现，复用）：
- `Q = diag([15,15,20, 0,0,0,0, 8,8,15])` — pos/quat/vel 权重
- `R_energy = diag([3, 50, 50, 30])` — fc/ω 控制代价（含能量罚）
- 已暴露：`lambda_energy`（能量权重增量）、`q_xy_scale`（xy 跟踪缩放）、`energy_first`（能耗优先开关）
- 组件：TCWP（风预测）、CMA、PredictiveEnergyManager、WindUtilizationAdvisor

### 3.2 重设计目标（必须实现的接口）

#### (R1) 代价权重运行时可注入

```python
class MPCController:
    def set_cost_weights(self, theta: dict):
        """RL 动作接口。线程安全, 50Hz 控制循环中途可热更新。
        theta = {
            'q_xy':       float,   # 水平位置跟踪权重
            'q_z':        float,   # 高度跟踪权重
            'lambda_energy': float,# 能耗权重 (加到 R 对角)
            'r_omega':    float,   # 角速率控制代价
        }
        所有键可选; 缺省保持当前值。带 clip 到安全区间。
        """
```

要求：
- 权重更新**不重建求解器**（只改 Q/R 对角），避免 50Hz 循环卡顿
- 每个权重 **clip 到 [θ_min, θ_max]**，RL 给出越界值也不会让 MPC 失稳
- 记录每步实际生效的 θ 到 CSV（可复现 + 论文画图）

#### (R2) 能耗作为可观测输出

```python
    def get_power_estimate(self) -> float:
        """当前 BEMT 功耗 [W], 供 RL 奖励 + 日志。"""
    def get_predicted_energy(self) -> float:
        """预测时域内累积能量 [J], 供 MPC 自身能耗优化。"""
```

#### (R3) 可微性决策 —— **暂定免梯度（OSQP/iLQR 黑盒 + PPO）**

| 方案 | 适用 | 取舍 |
|------|------|------|
| **A. 黑盒 MPC + PPO**（推荐起步） | 低频权重调度 | 实现简单、上实机稳、无需可微求解器；RL 样本效率略低 |
| B. Differentiable MPC + 梯度 | 高频权重学习 | 样本效率高；需 acados/theseus/diff-MPC 重写，复杂度高 |

**决策：先走 A**，与"能耗优先 + 低频调度 + 上实机"目标一致。
保留 B 作为后续升级（论文可作为 future work 或 ablation）。

#### (R4) 与 plant / PX4 接口不变

- 输出仍为 `AttitudeTarget` (body rate + thrust, `setpoint_raw/attitude`)
- 兼容现有 HIL（plant_6dof）与 SITL（Gazebo）

### 3.3 重设计交付物

1. `mpc_controller.py` — 满足 R1–R4 的重构控制器（从现有 mpc_node 抽离控制逻辑，与 ROS 节点解耦，便于 RL 训练时离线调用）✅
2. `mpc_node.py` — 瘦身为 ROS 包装层 ✅ (660→~290 行)
3. 单元测试：权重热更新不崩、越界 clip、能耗输出正确 ✅ `test_mpc_controller.py` 28/28 通过

**✅ M1 完成 (2026-06-28)**
- `mpc_controller.MPCController`：无 ROS 依赖，RL 可直接 import 调用 `step()`
- R1 `set_cost_weights(theta)`：线程安全，原地改 Q/R 对角（iLQR 持同一引用），不重建求解器；clip 到 `WEIGHT_BOUNDS`；nan/inf/未知键安全丢弃
  - `WEIGHT_BOUNDS = {q_xy:(0.5,60), q_z:(0.5,80), lambda_energy:(0,80), r_omega:(5,200)}`

## 4. 功耗模型（节能度量 —— sim-to-real 一致性核心）

### 4.1 现状：两套并存（需统一）

| 模型 | 输入 | 位置 | 用途 |
|------|------|------|------|
| `UnifiedPowerEstimator` | IMU 加速度 → 推力 → P∝T^1.5 | power_model.py | PID/MPC 实时估算 |
| `bem_power_10d` | 总推力 f_c + 姿态 + 风 → BEMT | quadrotor_dynamics.py | MPC 内部能耗 |

### 4.2 目标：单一权威功耗定义

**问题**：仿真拿不到真 RPM（HIL 是归一化电机指令 0~1），实机靠 ESC DShot RPM 遥测。
若 sim 和 real 用不同功耗定义，RL 学到的"节能策略"到实机失真。

**方案：BEMT 为唯一权威模型，输入侧做 sim/real 适配层**

```
仿真:  电机指令(0~1) ──[电机模型]──▶ ω_rotor(rad/s) ─┐
实机:  ESC DShot RPM ──[单位换算]──▶ ω_rotor(rad/s) ─┼──▶ BEMT ──▶ P_total[W]
                                                      ┘   (同一函数)
```

要求：
- 定义统一函数 `power_from_rotor_speeds(omegas: np.ndarray[4]) -> float`
- 仿真侧：plant_6dof 已有 `_compute_rotor_omegas()`，直接喂入
- 实机侧：订阅 `/mavros/esc_telemetry` 或 `actuator_outputs` + 电机模型
- **能量积分**：`E = ∫P dt`，每回合报告总能量 (Wh / J)
- 论文方法论章节明确写出 sim/real 功耗一致性策略（可复现性加分）

### 4.3 待办

- [ ] 标定 BEMT 参数到 Iris（thrust_scale、诱导功率系数），用 plant_6dof 悬停功耗对齐
- [ ] 实机侧确认 ESC 是否支持 RPM 遥测（决定实机功耗来源）
- [ ] 统一接口 `power_from_rotor_speeds`，sim/real 共用

### 4.4 能耗一致性与循环论证防御（审稿人答辩 — 对本项目最致命）

> 本项目核心 claim 是“能耗最优姿态调节”。审稿人会直接攻击：
> **“一个看不见转子气动/诱导功率/废阻的 MPC 简化模型，凭什么能找到能耗最优姿态？**
> **你在用一个看不见能量来源的模型去优化能量。”**

#### 风险：循环论证（circular reasoning）
若 MPC “优化能耗”用的模型与“评测能耗”的模型是同一套简化模型 →
用错误模型优化、又用同套模型评测 → **省下的电是假的**。这是本项目最大的方法论险。

#### 防御：职责分工 + 能量权威与优化器解耦

| 角色 | 能量职责 | 用什么模型 |
|------|---------|----------|
| **MPC (L2)** | **不负责“理解”能量** — 只管可行性+跟踪+约束 | 简化 10D CTBR（OK） |
| **RL (L3)** | **提供能量智能** — 用真实测量功耗做奖励信号 | BEMT 实测 / 真机 V·I (ground truth) |
| **评测** | E_total 必须用**高保真 plant / 实测**，绝不用 MPC 内模 | power_model_v3 / 真机 |

干净论证：**MPC = 约束可行性，RL = 用 ground-truth 能量做调度**。
RL 看到的是独立权威 BEMT 实测功耗，与 MPC 内模无关 → 避开循环论证。

#### 必须拍板的不一致：MPC 内部 `lambda_energy` 的定性

问题：MPC 内部那个 `lambda_energy`（加在推力代价 R[fc]）算不算“能量优化”？
- 若算 → 它基于看不见废阻/诱导功率的简化模型，与 BEMT 权威模型矛盾 → 被抓。
- **定论（本项目采纳）：MPC 的 `lambda_energy` 只是一个“行为旋钮”(behavior knob)，
  不是忠实能量模型；真正的能量权威是 RL 看到的 BEMT 实测功耗。**
  该定论与“RL 调权重”的整体叙事自洽，且消除循环论证嫌疑。
- 论文方法论章节须明写此定性，不留歧义。

#### 极端湍流下“风当外部扰动”的边界
时间尺度分离 + “风当扰动 d_seq”在温和风下成立；极端湍流下气动耦合可能是一阶效应。
防御：风扰**已注入预测**（`compute_wind_disturbance_10d`进 d_seq）+ TCWP 在线预测 + 内环吸收；
需 §6.3 极端档实验证据（跟踪/能耗有界且优于 baseline）为简化辩护。

## 6. 对照实验设计（SCI 标准）

### 6.0 原理验证：能耗-姿态关系（motivation 实验）

> ⚠️ **2026-06-28 离线验证结论（纠正了初始设想）**：
> “抹风定点悬停的能耗最优倾角”**不存在**。纯刚体定点(v=0)下，
> 力平衡 `tan(φ)=F_drag/mg`、`T=√(F_drag²+(mg)²)` 唯一锁死倾角与推力，
> 倾角**非自由变量**，扫不出 U 形。原始“迎风倾斜省电”直觉在严格定点约束下不成立。

**修正后的物理事实（真实、教科书级）：**
节能的自由度来自**是否允许漂移/前飞**，而非倾角本身：
- 经典前飞 power curve：最优空速 v*≈7 m/s 前飞比悬停**省 ~22%**
  （133.6W vs 172.2W，配平倾角 5.9°）。诱导功率↓ + 废阻功率↑ = U 形。
- 对**定点抗风**任务：节能空间来自“顺风方向小幅位移借风势”
  （降相对气流→降诱导功率），而非死守原点。这正是 mpc_components 里
  `WindUtilizationAdvisor`（风能利用）的思路。节能 vs 精度是真实 trade-off。

**motivation 实验（修正后）：**
- (a) **前飞 power curve**：扫空速测配平功耗 → 经典 U 形，谷底 v*
- (b) **死守原点 vs 借风位移**：同一风场下，对比严格定点与允许有界位移的能耗
  → 证明节能-精度 Pareto 前沿存在，为 RL 调权重提供探索空间

**🔴 阻塞性前置（功耗模型缺陷，必须在 RL 之前修）：**
现有 `bem_power_10d` **缺废阻功率项** `P_par = D·v_air = 0.5ρ·CdA·v³`，
且未接入已有的 `effective_CdA(roll,pitch)`。导致功耗对相对气流**单调递减**，
看不到“前飞/借风省电”。不修则 RL 节能奖励被误导 → 学出错误策略。
→ **M2 功耗统一时必须补 P_par + 接 effective_CdA，并用 power curve 标定验证。**

**交付物：**`tilt_sweep_offline.py`（已完成，离线验证）；HIL 前飞标定实验（随 M2 功耗修复）。

### 6.2.1 架构对等性与精度定位（审稿人答辩 — 关键）

> **核心立场：精度是"约束"，能耗是"目标"。不在定点抗扰精度上击败 PID，也不应以此为目标。**

**对照的架构不对等（必须向审稿人讲清，否则被攻击）：**
- **B0 纯 PID 走"范式 A"**：PID 输出位置/速度 setpoint → 经 PX4 **完整四层级联**
  （位置 P → 速度 PID → 姿态 P → 角速率 PID），每层经十余年工程打磨且带积分。
- **B1–B3 MPC 走"范式 C"（CTBR）**：MPC 输出 `[f_c, ω]` → **仅经 PX4 最内层角速率 PID**，
  位置/速度/姿态三层由我们的单层 MPC 替代。这是 agile flight 的标准接口
  （Sun et al. TRO2022、Hanover RAL2021、px4-mpc 官方实现均同）。

**为何 MPC 在"定点抗随机湍流精度"上不具结构优势（实测佐证）：**
- MPC 的优势是**预测**；但随机湍流**不可预测**，阵风的下一秒无法前瞻 →
  MPC 在该场景退化为反应式控制。实测 CSV：风稳态下角速率命令**仅用满量程 24%**
  （ωx 峰 0.35 / 上限 1.49），即 MPC 并非"能力不足"，而是"无可预测信息可用"。
- 反应式调节正是 PX4 三层积分级联的主场。**用单层 MPC 在 PID 主场比精度，先天不对等。**

**因此精度的论文定位 = "达到与 PID 可比的量级（约束满足、不发散）"，而非"超越 PID"。**
真正展示 MPC/RL 价值的维度是**能耗**（PID 无能量目标）与**多目标自适应**（RL 在线调权重，PID 不能）。

**叙事公式（论文结论句）：**
> "在精度约束均满足（同量级、不发散）的前提下，能耗 B3 < B2 < B1 < B0，
> 且 B3 在变风况下自适应能力最强。" —— 精度够用即可，能耗与自适应才是胜场。

**对照关系的职责划分：**
- **B3 vs B1** = RL 调度的增量价值（核心消融，主结论）
- **B3/B1 vs B0** = 相对工业标准方法的意义（证明研究值得做；B0 是"绝对锚点"非"精度擂台"）
- **B3 vs B2** = RL 优于启发式规则（堵"if-else 就够了"的质疑）

### 6.2.2 B1 固定权重 MPC 的抗扰设计（已实现，2026-06-29）

B1（也是 B2/B3 的 MPC 内核）的抗扰能力来自三项设计，均有文献背书（审稿人友好）：

| 设计 | 内容 | 文献依据 |
|------|------|---------|
| **姿态约束** | 姿态参考=水平（`euler_to_quat(0,0,yaw)`）+ Q 四元数加权 | NMPC-QUA；Sun TRO2022 |
| **终端代价** | `P_terminal` 位置×15/速度×6（对齐 px4-mpc 官方 30/10） | 最优控制教科书；px4-mpc |
| **offset-free MPC** | 增广扰动加速度 `d_acc` 观测器（速度残差型，三轴），MPC 预测内补偿外力/模型失配 | Pannocchia & Rawlings 2003；Hanover RAL2021 |

**演进实测（HIL，7.5 m/s 湍流，seed=42）：**

| 阶段 | XY RMSE | Z RMSE | 备注 |
|------|---------|--------|------|
| 姿态放任（Q_quat=0） | **发散 136 m** | 坠地 | 姿态环被绕过又无人管 → 侧翻 |
| + 姿态约束（NMPC-QUA） | 0.9 m | — | 有界 |
| + 终端代价 | 0.91 m | 0.34 m | 高度稳 |
| + offset-free（仅水平） | 0.48 m | 0.55 m | XY 砍半（偏置被消） |
| **+ offset-free（含 z 轴）** | **0.49 m** | **0.13 m** | Z 砍 76%（补倾斜推力亏空） |
| 对照 B0 纯 PID | 0.08 m | — | 同量级，PID 主场 |

> **关键诊断（决定不引入 acados）**：0.9 m 漂移 = 偏置 ~0.55 m + 震荡 ~0.3 m。
> acados（长时域 + 大 R_ω）只治震荡，不治偏置；偏置根源是缺积分 + 模型失配，
> 已由 offset-free 直接消除（XY 砍半）。换求解器对主要矛盾无益，故不采用。
> 短时域是纯 Python iLQR 算力约束（N=15 已占满 50 Hz 预算），R_ω≤50。

### 6.4 评测指标

**主指标（能耗优先）：**
- `E_total` 每回合总能量 [Wh] ← **核心**
- `P_mean` 平均功率 [W]
- `E_per_error` 单位精度能耗 = E_total / (误差预算内成功率)

**约束/精度指标：**
- 位置误差 RMSE / p95 / max [m]
- 姿态 RMSE, 倾角峰值 [deg]
- 约束违反次数, 任务成功率 (误差始终 < 阈值)

**鲁棒性指标：**
- 控制抖动 (jerk RMS), 权重抖动 ‖Δθ‖（RL 专属）

### 6.5 统计严谨性（SCI vs demo 的分水岭）

- 每组每风况 **N≥10** 独立 seed → 报告 **mean ± std**
- **显著性检验**：B3 vs B0/B1/B2 做配对检验（**Wilcoxon signed-rank**，不假设正态；或配对 t-test）
- 报告 **p-value** 与 **效应量** (Cohen's d / 相对提升 %)
- **Pareto 前沿图**：能耗 vs 位置误差，展示 B3 占优区域 ← 论文核心图
- 同一 seed 跨组公平（wind_ab.py 已是此设计）

### 6.6 消融子实验（论文 ablation 章节）

- 奖励项消融：去掉 α_E / α_s 各跑一遍，证明能耗项与平滑项必要性
- 观测消融：去掉风估计 ŵ，证明风感知的价值
- 权重维度消融：只调 λ_energy vs 调全部 θ

### 6.7 Sim-to-real

- 同一 B3 策略：sim 训练 → 实机部署（不重新训练，或仅少量微调）
- 报告 **sim-to-real gap**：实机 E_total / 误差 相对 sim 的偏差 %
- 域随机化策略（plant 参数 ±X%）在训练中的作用（ablation）
