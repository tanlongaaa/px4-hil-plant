## 1. 系统架构

### 1.1 三层分层控制

```
┌──────────────────────────────────────────────────────────────┐
│  L3  RL 策略 (Cost-Weight Scheduler)          ~1–5 Hz (低频)   │
│      输入: 风估计 ŵ, 姿态, 位置误差, 功耗, 湍流强度σ̂           │
│      输出: MPC 代价权重 θ = {Q_xy, λ_energy, R_ω, ...}         │
│      训练: PPO (MPC 当黑盒) → 后续可升级 differentiable MPC     │
└───────────────────────────────┬──────────────────────────────┘
                                 │ θ (cost weights)
                                 ▼
┌──────────────────────────────────────────────────────────────┐
│  L2  MPC (Trajectory-Coherent Wind-Adaptive MPC)   50 Hz      │
│      模型: 10D CTBR 刚体 + 风扰 + BEMT 能耗                     │
│      求解: iLQR / OSQP, 含约束 (推力/角速率/倾角)               │
│      输出: CTBR = [f_c, ωx, ωy, ωz]  (总推力 + 体轴角速率)      │
│      角色: 安全滤波器 — 保证可行性与约束, RL 永不直接碰执行器   │
└───────────────────────────────┬──────────────────────────────┘
                                 │ AttitudeTarget (setpoint_raw/attitude)
                                 ▼
┌──────────────────────────────────────────────────────────────┐
│  L1  PX4 底层 (角速率/姿态 PID + 混控器)        ~250–1000 Hz   │
│      吸收: 电机滞后, 高频扰动                                   │
│      已标定: MPC_XY_VEL_I_ACC=1.2 等 (抗风调参完成)            │
└───────────────────────────────┬──────────────────────────────┘
                                 │ motor outputs (0~1) / 实机 PWM/DShot
                                 ▼
┌──────────────────────────────────────────────────────────────┐
│  Plant  仿真: plant_6dof (HIL) ──┐    实机: 真实四旋翼         │
│         风场: wind_field (Dryden)│    ESC RPM 遥测 → 功耗      │
│         功耗: BEMT (电机指令→RPM→P)                            │
└──────────────────────────────────────────────────────────────┘
```

### 1.2 分层理由（论文 justification）

| 层 | 频率 | 为什么是它 |
|----|------|-----------|
| RL 调权重 | 低频 | 安全（MPC 兜底约束）、可解释、易部署、对实机鲁棒；避免高频 RL 直接控制的安全风险 |
| MPC | 50 Hz | 在线重规划吸收可预测扰动；显式约束保证可行性；代价权重是自然的"行为旋钮" |
| PX4 PID | 高频 | 吸收电机滞后与不可建模高频；工业级成熟、直接对应实机 |

**核心论点（contribution）：**
> 现有抗风控制几乎都以**跟踪精度**为目标。本工作提出**能耗优先**的视角：
> RL 学习**风况自适应的 MPC 代价权重**，在「能耗 vs 精度」Pareto 前沿上，
> 依据实时风态选择能耗最优工作点。MPC 保证安全可行，RL 只在外层调度。

### 1.3 与 ACMPC 的区别（novelty 定位）

| 维度 | ACMPC (TRO 2025) | 本工作 |
|------|------------------|--------|
| RL 学习对象 | 端到端策略 + 可微 MPC | **MPC 代价权重调度** |
| 优化目标 | 跟踪精度 | **能耗优先** + 精度约束 |
| 仿真保真度 | 纯 PyTorch 刚体, 无电机/气动 | **HIL plant_6dof + 真 PX4** (含风扰/旋翼气动) |
| 底层 | Agilicious 角速率 PID (闭源) | **PX4 PID (开源, 已标定)** |
| 能耗模型 | 无 | **BEMT, sim/real 统一** |
| 部署 | 实机 | 实机 + sim-to-real gap 量化 |

## 2. 问题形式化定义

### 2.1 控制对象（被控系统）

四旋翼 10D CTBR 状态（与现有 `quadrotor_dynamics_10d` 对齐）：

```
x = [p(3), q(4), v(3)] ∈ R^10     # 位置(ENU) + 姿态四元数 + 速度
u = [f_c, ωx, ωy, ωz] ∈ R^4       # 总推力(N) + 体轴角速率指令 (CTBR)
```

风扰 `w ∈ R^3`（ENU 风速），经 plant 转子级 + 机身级气动注入。

### 2.2 RL 问题（外层 MDP）

| 元素 | 定义 | 说明 |
|------|------|------|
| **观测 o_t** | `[ê_pos(3), v(3), tilt(2), ŵ(3), σ̂_turb(1), P_est(1), θ_{t-1}]` | 风估计 + 误差 + 功耗 + 上一步权重 |
| **动作 a_t** | `θ_t = [Q_xy, Q_z, λ_energy, R_ω]`（归一化到有界区间） | MPC 代价权重子集 |
| **奖励 r_t** | 见 §2.3 | 能耗优先的复合奖励 |
| **频率** | 1–5 Hz（每 N 个 MPC 步更新一次 θ） | 低频调度 |
| **回合** | 一次定点抗风飞行（固定时长 T，固定风 seed） | |

### 2.3 奖励函数（能耗优先）

```
r_t = - α_E · (E_t / E_hover)              # 能耗项 (主) — 归一化到悬停能耗
      - α_p · ‖e_pos‖² · 1[‖e_pos‖ > δ]    # 位置误差 (仅超出容差 δ 才罚)
      - α_c · (constraint_violation)         # 约束违反 (硬惩罚)
      - α_s · ‖θ_t − θ_{t-1}‖²              # 权重平滑 (防抖动/振荡)
```

设计要点：
- **能耗优先**通过 `α_E ≫ α_p` 体现，且位置误差用**死区**（容差 δ 内不罚）→ 鼓励"够用就好"
- 约束违反走 MPC 硬约束 + 奖励软惩罚双保险
- 平滑项防 RL 高频抖权重激发系统振荡

> ⚠️ 待定参数：α_E, α_p, α_c, α_s, δ, E_hover 基准。
> 这些是**奖励塑形**关键，需在 §6 实验中通过敏感性分析确定，并在论文中报告。

### 2.4 约束（MPC 硬约束，安全边界）

```
f_c ∈ [2, 20] N              # 总推力 (10D nominal, 见现有代码)
|ωx|, |ωy| ≤ 0.1 rad/s       # 体轴角速率 (CTBR, 极小, 防超调)
|ωz| ≤ 0.05 rad/s
tilt ≤ MPC_TILTMAX_AIR=45°   # 倾角 (PX4 层)
```

## 3. MPC 接口规范（重设计要求）

> 这是 MPC 重设计的**验收标准**。重设计前必须满足以下接口，否则 RL 无法接入。

### 3.1 现状（已有，offboard_test/scripts/）

现有 `mpc_node.py` 已是 10D CTBR iLQR，且**已有权重钩子**（关键发现，复用）：
- `Q = diag([15,15,20, 0,0,0,0, 8,8,15])` — pos/quat/vel 权重
- `R_energy = diag([3, 50, 50, 30])` — fc/ω 控制代价（含能量罚）
- 已暴露：`lambda_energy`（能量权重增量）、`q_xy_scale`（xy 跟踪缩放）、`energy_first`（能耗优先开关）
- 组件：TCWP（风预测）、CMA、PredictiveEnergyManager、WindUtilizationAdvisor

### 3.2 重设计目标（必须实现的接口）

#### (R1) 代价权重运行时可注入

```python
class MPCController:
    def set_cost_weights(self, theta: dict):
        """RL 动作接口。线程安全, 50Hz 控制循环中途可热更新。
        theta = {
            'q_xy':       float,   # 水平位置跟踪权重
            'q_z':        float,   # 高度跟踪权重
            'lambda_energy': float,# 能耗权重 (加到 R 对角)
            'r_omega':    float,   # 角速率控制代价
        }
        所有键可选; 缺省保持当前值。带 clip 到安全区间。
        """
```

要求：
- 权重更新**不重建求解器**（只改 Q/R 对角），避免 50Hz 循环卡顿
- 每个权重 **clip 到 [θ_min, θ_max]**，RL 给出越界值也不会让 MPC 失稳
- 记录每步实际生效的 θ 到 CSV（可复现 + 论文画图）

#### (R2) 能耗作为可观测输出

```python
    def get_power_estimate(self) -> float:
        """当前 BEMT 功耗 [W], 供 RL 奖励 + 日志。"""
    def get_predicted_energy(self) -> float:
        """预测时域内累积能量 [J], 供 MPC 自身能耗优化。"""
```

#### (R3) 可微性决策 —— **暂定免梯度（OSQP/iLQR 黑盒 + PPO）**

| 方案 | 适用 | 取舍 |
|------|------|------|
| **A. 黑盒 MPC + PPO**（推荐起步） | 低频权重调度 | 实现简单、上实机稳、无需可微求解器；RL 样本效率略低 |
| B. Differentiable MPC + 梯度 | 高频权重学习 | 样本效率高；需 acados/theseus/diff-MPC 重写，复杂度高 |

**决策：先走 A**，与"能耗优先 + 低频调度 + 上实机"目标一致。
保留 B 作为后续升级（论文可作为 future work 或 ablation）。

#### (R4) 与 plant / PX4 接口不变

- 输出仍为 `AttitudeTarget` (body rate + thrust, `setpoint_raw/attitude`)
- 兼容现有 HIL（plant_6dof）与 SITL（Gazebo）

### 3.3 重设计交付物

1. `mpc_controller.py` — 满足 R1–R4 的重构控制器（从现有 mpc_node 抽离控制逻辑，与 ROS 节点解耦，便于 RL 训练时离线调用）
2. `mpc_node.py` — 瘦身为 ROS 包装层
3. 单元测试：权重热更新不崩、越界 clip、能耗输出正确

## 4. 功耗模型（节能度量 —— sim-to-real 一致性核心）

### 4.1 现状：两套并存（需统一）

| 模型 | 输入 | 位置 | 用途 |
|------|------|------|------|
| `UnifiedPowerEstimator` | IMU 加速度 → 推力 → P∝T^1.5 | power_model.py | PID/MPC 实时估算 |
| `bem_power_10d` | 总推力 f_c + 姿态 + 风 → BEMT | quadrotor_dynamics.py | MPC 内部能耗 |

### 4.2 目标：单一权威功耗定义

**问题**：仿真拿不到真 RPM（HIL 是归一化电机指令 0~1），实机靠 ESC DShot RPM 遥测。
若 sim 和 real 用不同功耗定义，RL 学到的"节能策略"到实机失真。

**方案：BEMT 为唯一权威模型，输入侧做 sim/real 适配层**

```
仿真:  电机指令(0~1) ──[电机模型]──▶ ω_rotor(rad/s) ─┐
实机:  ESC DShot RPM ──[单位换算]──▶ ω_rotor(rad/s) ─┼──▶ BEMT ──▶ P_total[W]
                                                      ┘   (同一函数)
```

要求：
- 定义统一函数 `power_from_rotor_speeds(omegas: np.ndarray[4]) -> float`
- 仿真侧：plant_6dof 已有 `_compute_rotor_omegas()`，直接喂入
- 实机侧：订阅 `/mavros/esc_telemetry` 或 `actuator_outputs` + 电机模型
- **能量积分**：`E = ∫P dt`，每回合报告总能量 (Wh / J)
- 论文方法论章节明确写出 sim/real 功耗一致性策略（可复现性加分）

- [ ] 标定 BEMT 参数到 Iris（thrust_scale、诱导功率系数），用 plant_6dof 悬停功耗对齐
- [ ] 实机侧确认 ESC 是否支持 RPM 遥测（决定实机功耗来源）
- [ ] 统一接口 `power_from_rotor_speeds`，sim/real 共用

## 6. 对照实验设计（SCI 标准）

### 6.0 原理验证：最优倾角扫描（motivation 实验）

**目的：**用数据证明“姿态能省能”真实存在，为 RL+MPC 的必要性提供正当性。论文开篇 motivation 图。

**方法：**固定横风（多个风速：4 / 8 / 12 m/s），让无人机在一系列**固定倾角**下悬停
（0°,2°,4°,...,扫到接近 MPC_TILTMAX），每个倾角测稳态总功耗。画 P-vs-倾角曲线。

**预期结果：**U 形曲线，存在能耗最优倾角 θ*（推力矢量对齐“重力+气动阻力”合力）。
且 θ* 随风速增大 → 证明需要**自适应** → RL+MPC 的存在理由。

**与本方法关系：**U 形谷底就是 B3 理论上要自动找到的工作点。
可把 B3 实际飞出的倾角叠在这张图上，展示它收敛到 θ* 附近。

**交付物：**`tilt_sweep.py`（复用 HIL + 风场 + BEMT，改脚本扫倾角）；P-vs-tilt-vs-windspeed 图。
⚠️ 注：固定倾角悬停需要能绕过 PX4 位置环直接指定姿态设定值（或用外环补偿偏置），
实现时需确认怎么在抗风悬停的同时强制固定倾角（可能需姿态直驱 + 位置软约束）。

### 6.1 任务场景

| 任务 | 描述 | 主要用途 |
|------|------|---------|
| **T1 定点抗风悬停** (station-keeping) | 固定点 (0,0,2.5) 抗风 T=60s | 能耗最干净量化, 主实验 |
| **T2 轨迹跟踪** (圆/8字) | 跟踪参考轨迹 | 证明泛化能力 |

### 6.2 对照组（ablation，逐层证明贡献）

| 组 | 控制器 | 证明什么 |
|----|--------|---------|
| **B0** | 纯 PX4 PID (I=1.2, 已调) | 最底基准 |
| **B1** | 固定权重 MPC + PID | MPC 相对纯 PID 的增益 |
| **B2** | 启发式调权重 MPC + PID（规则: 按风速分档调 λ_energy） | RL 相对手工规则的增益 |
| **B3** | **RL 调权重 MPC + PID（本方法）** | 主角 |

> 逐组只加一个变化 → 干净的 ablation，审稿人能清楚看到每层贡献。

### 6.3 风况矩阵（标准化、可复现）

| 维度 | 取值 | 说明 |
|------|------|------|
| 湍流强度 sigma_scale | 0.25 (轻) / 0.5 (中) / 1.0 (极端) | 覆盖到极端 |
| 平均风速 u_ref | 5 / 8 / 12 m/s | |
| 随机种子 seed | {1..10} (固定 10 个) | **同 seed 喂给所有对照组** |
| 阵风 gust | 含 / 不含 | |

每个 (控制器 × 风况) 组合跑 **N=10** 次独立 seed。

### 6.4 评测指标

**主指标（能耗优先）：**
- `E_total` 每回合总能量 [Wh] ← **核心**
- `P_mean` 平均功率 [W]
- `E_per_error` 单位精度能耗 = E_total / (误差预算内成功率)

**约束/精度指标：**
- 位置误差 RMSE / p95 / max [m]
- 姿态 RMSE, 倾角峰值 [deg]
- 约束违反次数, 任务成功率 (误差始终 < 阈值)

**鲁棒性指标：**
- 控制抖动 (jerk RMS), 权重抖动 ‖Δθ‖（RL 专属）

### 6.5 统计严谨性（SCI vs demo 的分水岭）

- 每组每风况 **N≥10** 独立 seed → 报告 **mean ± std**
- **显著性检验**：B3 vs B0/B1/B2 做配对检验（**Wilcoxon signed-rank**，不假设正态；或配对 t-test）
- 报告 **p-value** 与 **效应量** (Cohen's d / 相对提升 %)
- **Pareto 前沿图**：能耗 vs 位置误差，展示 B3 占优区域 ← 论文核心图
- 同一 seed 跨组公平（wind_ab.py 已是此设计）

### 6.6 消融子实验（论文 ablation 章节）

- 奖励项消融：去掉 α_E / α_s 各跑一遍，证明能耗项与平滑项必要性
- 观测消融：去掉风估计 ŵ，证明风感知的价值
- 权重维度消融：只调 λ_energy vs 调全部 θ

### 6.7 Sim-to-real

- 同一 B3 策略：sim 训练 → 实机部署（不重新训练，或仅少量微调）
- 报告 **sim-to-real gap**：实机 E_total / 误差 相对 sim 的偏差 %
- 域随机化策略（plant 参数 ±X%）在训练中的作用（ablation）

### 关键技术决策（已定）

| 决策 | 选择 | 理由 |
|------|------|------|
